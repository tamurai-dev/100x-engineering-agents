# Task Planner ルーブリック

## タスク分解の粒度（0.0-1.0）
- 各サブタスクが 1 PR で完結できるサイズ
- 大きすぎず小さすぎない適切な粒度
- 各サブタスクが独立して実行可能

## 依存関係の明確さ（0.0-1.0）
- サブタスク間の依存関係が明示されている
- 依存関係が論理的に正しい
- 並行実行可能なタスクが識別されている

## 完了条件の具体性（0.0-1.0）
- 各サブタスクに Definition of Done が定義されている
- 完了条件が検証可能（曖昧な「完了した」ではなく「テストが PASS する」レベル）

## リスク特定（0.0-1.0）
- 技術的リスクが特定されている
- 不確実性の高い部分にスパイクタスクが設けられている
- リスクへの対策が提案されている

## 制約条件の考慮（0.0-1.0）
- fixture で提供された技術的制約（PostgreSQL, JWT, AWS ECS）が計画に反映されている
- 既存アーキテクチャとの整合性が考慮されている
