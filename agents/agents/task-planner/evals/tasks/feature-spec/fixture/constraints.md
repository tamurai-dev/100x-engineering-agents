# 技術的制約

## 既存技術スタック
- **バックエンド**: Node.js 20 + Express 4
- **データベース**: PostgreSQL 16（AWS RDS）
- **認証**: JWT（既存の auth ミドルウェアあり）
- **キャッシュ**: Redis 7（AWS ElastiCache）
- **メッセージキュー**: 未導入（候補: SQS, Redis Pub/Sub）
- **デプロイ**: AWS ECS Fargate（Docker コンテナ）
- **CDN**: CloudFront
- **CI/CD**: GitHub Actions

## チームの制約
- バックエンドエンジニア: 2名
- フロントエンドエンジニア: 1名
- 合計工数: 3スプリント（6週間）

## 技術的負債
- 現在の REST API にはバージョニングがない
- テストカバレッジ: 45%（目標 60%以上）
- 一部エンドポイントのレスポンスタイムが 500ms を超えている

## セキュリティ要件
- WebSocket 接続は JWT で認証すること
- 通知内容にユーザーの個人情報を含めないこと
- 通知の送信元を検証すること（なりすまし防止）
