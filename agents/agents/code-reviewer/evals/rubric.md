# Code Reviewer ルーブリック

## 指摘の具体性（0.0-1.0）
- 各指摘にファイルパスと行番号（または行範囲）が含まれている
- 修正方法が具体的で実装可能（「修正してください」ではなく「パラメータ化クエリを使用してください」レベル）
- 重要度分類（MUST FIX / SHOULD FIX / CONSIDER）が適切

## 網羅性（0.0-1.0）
- CRITICAL な問題を全て検出している
- セキュリティと品質の両面をカバーしている
- false positive（非問題の誤検出）が少ない

## 出力フォーマット（0.0-1.0）
- agent.md で定義した `[MUST FIX | SHOULD FIX | CONSIDER] ファイルパス:行番号` 形式に従っている
- 各指摘に「問題の説明」と「推奨される修正方法」が含まれている
- 日本語で明瞭に記述されている
