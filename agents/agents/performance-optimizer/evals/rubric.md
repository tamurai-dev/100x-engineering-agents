# Performance Optimizer ルーブリック

## 指摘の具体性（0.0-1.0）
- 各指摘にファイルパスと行番号（または行範囲）が含まれている
- パフォーマンス問題の計算量や影響度が定量的に説明されている
- 最適化方法が具体的で実装可能（「最適化してください」ではなく「JOIN で一括取得してください」レベル）
- 重要度分類（CRITICAL / HIGH / MEDIUM / LOW）が影響度に応じて適切

## 網羅性（0.0-1.0）
- CRITICAL なボトルネックを全て検出している
- 複数カテゴリ（アルゴリズム、DB、フロントエンド、メモリ等）を横断的にカバーしている
- false positive（実際には問題でないものの誤検出）が少ない

## 出力フォーマット（0.0-1.0）
- agent.md で定義した `[CRITICAL | HIGH | MEDIUM | LOW] <カテゴリ> ファイルパス:行番号` 形式に従っている
- 各指摘に「パフォーマンス問題の説明」「計算量/影響」「推奨される最適化方法」「期待される改善効果」が含まれている
- 日本語で明瞭に記述されている
