# 不動産仲介業務ワークフロー

## ワークフロー概要

不動産仲介会社の物件管理・顧客対応・契約管理の業務フロー。

### 1. 物件登録フロー
1. 物件オーナーから物件情報を受領
2. 担当者が物件情報をシステムに登録（所在地、間取り、価格、写真等）
3. 上長が物件情報を確認・承認
4. 承認後、物件をポータルサイトに公開

### 2. 問い合わせ対応フロー
1. 顧客がポータルサイトから物件に問い合わせ
2. 担当者に通知 → 初回連絡（24時間以内）
3. 顧客の希望条件をヒアリング → CRM に記録
4. マッチする物件を提案

### 3. 内覧・商談フロー
1. 内覧日程を調整（顧客・オーナー・担当者の3者）
2. 内覧実施 → 内覧レポート作成
3. 顧客の購入/賃借意向を確認
4. 条件交渉（価格・入居日等）

### 4. 契約フロー
1. 重要事項説明書の作成・交付
2. 契約書の作成
3. 契約締結（電子署名対応）
4. 決済・引渡し

### 5. アフターフォロー
1. 入居後の不具合対応
2. 契約更新管理（賃貸の場合）
3. 顧客満足度アンケート

## 関係者
- **物件オーナー**: 物件を所有・提供する側
- **顧客（買主/借主）**: 物件を探している側
- **営業担当者**: 物件紹介・内覧同行・契約手続き
- **上長（マネージャー）**: 承認・管理
- **管理者（システム管理）**: マスターデータ管理

## マルチテナント要件
- 複数の不動産仲介会社が同一システムを利用
- 会社間のデータは完全に分離

## 自動化候補
- 問い合わせ受領時の担当者自動アサイン
- 内覧日程の自動調整提案
- 契約更新時期のリマインド通知
- 月次業績レポートの自動生成
